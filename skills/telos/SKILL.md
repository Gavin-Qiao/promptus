# telos — scaffold the four stores

Initialize Promptus in a project, in dependency order. Telos first: every other store answers
to the direction it sets. Be **idempotent** — detect an already-initialized repo and never
clobber an existing store; report and stop instead.

## Procedure

1. **Telos first.** Copy `templates/TELOS.md` → `<repo>/TELOS.md` and fill it with the
   operator: the north star (what this project is trying to find out, and for whom) and any
   project-specific lines under the invariant. Nothing else makes sense without it; the scripts
   also locate the project root by finding `TELOS.md`.
2. **Ledger.** Copy `templates/ledger/RESEARCH-LEDGER.md` → `<repo>/ledger/RESEARCH-LEDGER.md`,
   keeping the NOW-header skeleton and the `<!-- kb:append-point -->` sentinel that `kb-add`
   targets. Fill the Mandate/Thesis/Guardrails from the Telos.
3. **Knowledge.** Create `docs/` and `docs/lit/` with their `INDEX.md` stubs (from `templates/`).
4. **Memory.** Create `memory/` with `MEMORY.md` (its sentinel + index header) from the template.
   Memory is per-project — each project owns what it has settled.
5. **Leave `.promptus/`** to be generated by `kb-index` (derived + gitignored). Add `/.promptus/`
   to the repo's `.gitignore` if absent.

## After scaffolding

Hand off to the `research-ledger` skill so recording starts immediately, and drop the
standing cadence into the project's `AGENTS.md` (template: `templates/AGENTS.md`). Then store
the first unit — the operator's mandate — as `kb-add --substrate ledger --kind DECISION`, and
run `bun "${CLAUDE_PLUGIN_ROOT}/scripts/kb-index.ts"` to prove the loop closes.

Pairs with the `/promptus-init` command, which runs this end to end against the current repo.
