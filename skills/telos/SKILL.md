# telos — scaffold the four stores

> **Status: STUB.** Contract below; body `TODO`.

## What this skill should do

Initialize Promptus in a project, in dependency order:

1. **Telos first.** Write `TELOS.md` from `templates/TELOS.md`: the project's north
   star + the invariant (rules that never bend). Nothing else makes sense without it.
2. **Ledger.** Create `ledger/RESEARCH-LEDGER.md` from the template, including the
   `<!-- kb:append-point -->` sentinel that `kb-add` targets.
3. **Knowledge.** Create `docs/` and `docs/lit/` with their index stubs.
4. **Memory.** Wire the memory store (resolve the global-vs-per-project decision in
   `TELOS.md` first).
5. Leave `.promptus/` to be generated by `kb-index` (it's derived + gitignored).

Pairs with the `/promptus-init` command, which runs this end to end in a target repo.
